  1. Other readings:
    • Object Methods: this ⇗
      • Object represent entities of the real world.
      • Here's an object literal: user = { sayHi() { alert( this.userName ); } };
      • Object's actions are represented in JS by functions stored as object properties.
        • For example, we can use a Function Expression to create a function and assign it to the property user instance as user.sayHi = function() { ... };, and then make a call to it as user.sayHi();.
      • A function that is a property of an object is called its method.
      • The value of this is the object instance “before the dot”, the one used to call the method, or the one that called the method. user.sayHi(); prints out the user name.
        • Methods allow objects to “act” like object.doSomething()
      • In JavaScript, keyword this behaves unlike most other programming languages. It can be used in any function, even if it’s not a method of an object. The value of this is evaluated during the run-time, depending on the context.
      • The concept of run-time evaluated this has both pluses and minuses. On the one hand, a function can be reused for different objects. On the other hand, the greater flexibility creates more possibilities for mistakes.
        • When a function is declared, it may use this, but that this has no value until the function is called.
        • A function can be copied between objects.
        • When a function is called in the “method” syntax: object.method(), the value of this during the call is object.
        • Note that arrow functions are special: they have no this. When this is accessed inside an arrow function, it is taken from outside.
      • Example on chaining : 
        
let ladder = { step: 0,
	up() {
		this.step++;
		return this;
	},
	down() {
		this.step--;
		return this;
	},
	showStep() {
		alert( this.step );
		return this;
	}
};

ladder.up().up().down().up().down().showStep();
    • "This" in Javascript ⇗:
      • this is a keyword whose value changes depending on how a function gets called. There six different ways where this can take on new values. They are with their references:
        1. Global context → Browser Window object
        2. Object construction → Object instance
        3. An object method → Object, then object instance
        4. A simple function → Browser Window object
        5. An arrow function → Always the same as this in the immediate scope
        6. An event listener → Set to the element that fired the event in the event listener
      • Make a variable self or that in case you need an auxiliary variable for storing this, at some scenario.
      • bind, call and apply can also be use to change the value of this within the function.
      • bind() is a method that is present in every function. It allows you to change the this context. This method takes in any number of arguments and returns the bound function. The first parameter you pass into bind becomes this in the bound function. Once you have created a bound function, you can call it anytime you wish. Example: const boundFunc = myFunction.bind( {someKey: 'someStringValue'}, // and any number of arguments...)
  2. JavaScript: Novice to Ninja:
    • Ch5: Objects:
      • Six primitive data types: strings, numbers, booleans, symbols, undefined, null, or object.
        • New PDT:
          • Symbol: Primitive which cannot be recreated nor mutated. const s1 = Symbol('Spiderman'); and const s2 = Symbol('Spiderman'); are completely different. They can be reused again and again by any object.
          • A new property can be added to an object using a symbol as a key if the square bracket notation is used.
      • Object types: built-in (array, function), and user defined.
      • Object Literals (OL): An object in JavaScript is a self-contained set of related values and functions. They act as a collection of named properties that map to any JavaScript value data type (see above). This is the preferred way of creating objects.
      • If a property’s value is a function, it is known as a method.
      • Calling Methods: superman.fly() or superman['fly']()
      • Objects are assigned by reference. This means that if a variable is assigned to an object that already exists, it will simply point to the exact same space in memory. So any changes made using either reference will affect the same object.
      • OL can be passed as a parameter to a function. This is useful when passing a "package of values/parameters". → greet( {greeting: `What's up dude`, age: 10, name: `Bart` } );
      • Object Literal Pattern or Namespacing: Objects containing several methods and properties, which can be used as small libraries, so not to confuse their value names with other value names from other namespaces, libraries or whatever. This is used for avoiding code clashes. Example, with namespace myMaths: 
        
	const myMaths = {
		square(x) {
			return x * x;
		},
		mean(array,callback) {
			if (callback) {
				array.map( callback );
			}
			const total = array.reduce((a, b) => a + b);
			return total/array.length;
		}
	};
      • Other built-in objects: They're global.
          Examples:
        • JSON (JavaScript Object Notation), portable data format.
        • Math, a number of mathematical constants, for mathematical calculations.
        • Date, for date objects.
        • RegExp, for Regular Expressions.
      • Using namespaces: Any references to the functions need to be replaced with namespace.function() outside the object, or this.function() inside the object.

    • Ch7: Events:
      • Event handlers/listeners are used to handle out events on the page. Every time a page is clicked, a callback function will be called, and this function is automatically passed an event object as a parameter that contains information about the event.
      • When thinking about object events consider: event properties, event methods, event types.
      • What kind of information does that event object contain? Examples: event.type, event.target, event.screenX, event.screenY, event.pageX, event.pageY, event.clientX, event.clientY, etc.
      • Examples and problems:
        1. Inline event handlers: Example ⇗ Do not use.
        2. Older event handlers: document.onclick = function() { console.log('clicked!') }; Do not use.
        3. Recommended event handlers → Event Listeners. They allow multiple functions to be attached independently to different events. DO use. Examples:
          • Attached to the global object (usually the whole browser window): addEventListener('click' () => alert('clicked!')); Example: html-1 ⇗ and js-1 ⇗
          • Attached to node objects: document.body.addEventListener("eventType", callbackFunction, thirdParameter); Example: html-2 ⇗ and js-2 ⇗
      • Event listeners are part of the DOM. DOM is an interface, that allows to use JS to interact with the webpage.
      • All nodes objects have event handler properties. That so: document.onclick = function() { console.log('Clicked!'); }
      • Events classification:
        1. Mouse events: click, dblclick, mousedown, mouseup, mouseover, mouseout, mousemove.
        2. Keyboard events: keydown (when is pressed and held down), keypress (when a character is printed to the screen), keyup (when is released).
        3. Touch events: touchstart, touchend, touchmove, touchenter, touchleave, touchcancel.
