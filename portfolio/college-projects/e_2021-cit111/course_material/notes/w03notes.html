Week 03 Outcomes

  • Learn the purpose of a database
  • Describe the different types of databases
  • Learn the different architectures and hardware of a database system
  • Learn about data storage and the different types of data (data types)
  • Become familiar with SQL
  • Understand the different between DDL and DML statements

    (W03 - ERD and Datatypes Intro):

    Online Draft
  • Show the ERD that goes with the data they collected the first week of class.
  • Show the students how the data they had in their spreadsheet might look as tables in the database by looking at the ERD you will use. Look at the datatypes you chose and why.     (W03 - SQL Overview Practice)

  • Use the following datawinter database to demonstrate an overview of SQL.
    • datagathered_dump.sql
  • When showing the sql code remember, the students do not need to be able to create any of this code. The purpose is just to show them the code and explain what each does and the purpose of each. Let them know we will learn how to code later in our course. You can let the students download the dump file (W03 - Import datagathered.sql) and practice with it as you show them the statements. I typed the first few with them and went over common syntax errors like forgetting the semi-colon between statements. Also showing how to run just one statement at a time in Workbench. The queries that use aggregates, group bys, and joins I just showed them and didn't have them type them in. The point is to show how detailed of questions we can ask the query, not to teach the SQL statements at this point.
  • This is also a good chance to talk about datatypes and how they allow different data to be entered. For example, the worry_category is VARCHAR and the dist_category followed the categorization of the data gather sheets with ENUM. The query showing the top worries has a result set that is not as succinct since the data entry person wasn't limited on what they could enter. There are misspellings and categories that didn't fit with the worry_category as represented in the data gathering sheet.
